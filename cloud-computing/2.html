Topik 1 · Analogi
Analogi: sewa rumah vs bangun sendiri
Sebelum masuk ke istilah teknis, mari pikirkan model cloud seperti pilihan tempat tinggal.
On-premise = bangun rumah sendiri
Anda beli tanah, beli material, bangun, dan urus semua sendiri. Bebas tapi mahal dan butuh waktu lama.
IaaS = sewa tanah + bangunan kosong
Penyedia menyediakan struktur dasar. Anda isi sendiri, cat sendiri, atur furnitur sendiri.
PaaS = sewa apartemen semi-furnished
Dapur, listrik, air sudah ada. Anda tinggal bawa barang bawaan dan mulai hidup.
SaaS = tinggal di hotel
Semua sudah disiapkan. Anda tinggal masuk, pakai, dan bayar per malam. Zero konfigurasi.
Topik 2 · Model layanan
Tiga model layanan cloud
Semakin ke atas, semakin banyak yang dikelola penyedia — dan semakin sedikit yang perlu Anda urus.
SaaS — Software as a Service
Aplikasi siap pakai langsung dari browser
Penyedia kelola
semuanya
Contoh: Gmail, Microsoft 365, Zoom, Salesforce, Canva, Google Workspace
PaaS — Platform as a Service
Platform untuk membangun & deploy aplikasi
Anda kelola
aplikasi & data
Contoh: Google App Engine, Heroku, AWS Elastic Beanstalk, Vercel
IaaS — Infrastructure as a Service
Server virtual, storage, jaringan
Anda kelola
OS ke atas
Contoh: AWS EC2, Google Compute Engine, Azure Virtual Machines
On-premise (referensi)
Anda kelola
segalanya
Hardware, jaringan, OS, middleware, aplikasi, data — semua tanggung jawab Anda
Topik 3 · Tanggung jawab
Shared responsibility model
Siapa bertanggung jawab atas apa? Ini penting agar tidak ada yang "tidak dijaga" saat insiden terjadi.
Penyedia cloud bertanggung jawab atas:
  • Keamanan fisik data center
  • Hardware & infrastruktur jaringan
  • Hypervisor & virtualisasi
  • Ketersediaan layanan (uptime)
  • Keamanan platform itu sendiri
Pengguna bertanggung jawab atas:
  • Data yang disimpan di cloud
  • Manajemen akses & identitas (IAM)
  • Konfigurasi keamanan aplikasi
  • Enkripsi data sensitif
  • Kepatuhan regulasi (UU PDP, dll.)

Kesalahpahaman umum: "Data saya aman karena sudah di cloud." Penyedia mengamankan infrastrukturnya — tapi konfigurasi yang salah dari sisi pengguna tetap bisa menyebabkan kebocoran data.

Topik 5 · Model deployment
Model deployment cloud
Selain jenis layanan, cloud juga dibedakan berdasarkan di mana dan bagaimana infrastrukturnya dikelola.
Public cloud
Infrastruktur dimiliki dan dioperasikan penyedia. Dibagi bersama banyak pelanggan secara virtual.
Paling umum
Private cloud
Infrastruktur eksklusif untuk satu organisasi. Bisa di data center sendiri atau dikelola pihak ketiga.
Keamanan tinggi
Hybrid cloud
Kombinasi public dan private. Data sensitif tetap on-premise, beban kerja lain di public cloud.
Paling fleksibel
Multi-cloud
Menggunakan lebih dari satu penyedia cloud sekaligus. Menghindari vendor lock-in.
Strategi lanjutan

Bank Indonesia dan banyak BUMN memilih hybrid cloud — data nasabah tetap di server lokal sesuai regulasi OJK, sementara aplikasi digital banking berjalan di public cloud untuk skalabilitas.

Topik 6 · Panduan pemilihan
Bagaimana memilih model yang tepat?
Tidak ada satu jawaban yang benar untuk semua organisasi. Pertimbangkan faktor-faktor ini.
Pertanyaan Jika ya → pertimbangkan
Apakah tim punya kemampuan IT yang kuat? IaaS — kontrol lebih besar
Ingin fokus pada pengembangan aplikasi saja? PaaS — platform siap pakai
Butuh tools produktivitas langsung pakai? SaaS — zero konfigurasi
Ada regulasi ketat soal lokasi data? Hybrid atau Private cloud
Khawatir tergantung satu vendor? Strategi multi-cloud
Penutup · Ringkasan pertemuan 2
Yang kita pelajari hari ini
  • ✓ IaaS = infrastruktur virtual, PaaS = platform siap deploy, SaaS = aplikasi langsung pakai
  • ✓ Semakin ke SaaS, semakin sedikit yang perlu dikelola — tapi semakin terbatas kontrolnya
  • ✓ Shared responsibility: penyedia amankan infrastruktur, pengguna amankan data & konfigurasi
  • ✓ Public, private, hybrid, dan multi-cloud dipilih sesuai kebutuhan bisnis dan regulasi
  • ✓ Banyak layanan yang sudah Anda gunakan sehari-hari adalah SaaS — Gmail, Zoom, Microsoft 365

Pertemuan berikutnya: kita akan bedah virtualisasi dan container — teknologi yang jadi fondasi semua layanan cloud yang baru saja kita pelajari.
